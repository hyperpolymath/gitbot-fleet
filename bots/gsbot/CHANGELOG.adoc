= Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

== [0.2.0] - 2026-05-16

=== Changed
- *Full port from Python to Rust/SPARK.* The previous Python implementation
  has been removed in its entirety. No Python remains in the repository.
- Discord layer reimplemented on `poise` 0.6 over `serenity` 0.12 (prefix
  commands), replacing the former discord.py cogs. Each former cog is now a
  Rust module under `src/commands/`; command names, aliases, argument shapes,
  point awards and embed fields are preserved.
- Persistence reimplemented on `sqlx` 0.8 + SQLite, replacing the former
  SQLAlchemy ORM. Schema lives in `migrations/0001_init.sql` and is applied
  automatically at startup via `sqlx::migrate!`. SQLite only (no Postgres).
- Configuration via `dotenvy`; logging via `tracing` /
  `tracing-subscriber` / `tracing-appender`; async via `tokio`; errors via
  `anyhow` / `thiserror`.
- Tooling moved to cargo + a `Justfile` (`just build/test/run/lint/...`).
  Tests run with `cargo test --all-targets`; lint with `cargo clippy
  --all-targets -- -D warnings`; format with `cargo fmt`.
- Container build is now a multi-stage `Containerfile` (rust builder ->
  debian-bookworm-slim runtime, non-root `gsbot` user).

=== Added
- `src/domain.rs`: a pure, total, correctness-critical scoring kernel — the
  architecture's *SPARK seam*. Its numeric core is exported under a stable
  C ABI (`#[no_mangle] extern "C"`: `gsbot_material_overall_score`,
  `gsbot_lifespan_multiplier`, `gsbot_level_for_points`) so a
  formally-verified SPARK/Ada module can be linked in its place via the
  hyperpolymath Zig-FFI / Idris2-ABI pattern with no caller changes.
- Four binaries: `gsbot`, `gsbot-load-fixtures`, `gsbot-export-data`,
  `gsbot-backup-db`, plus the `gsbot` library crate.

=== Note
- This release is *behaviour-preserving*: the command surface
  (`sustainability`, `alternatives`, `care`, `tips`, `impact`, `compare`,
  `search`, `brands`, `profile`, `leaderboard`, `setpreference`, and the
  admin `loaddata`/`stats`/`announce`), point awards, ranks, scoring
  formulas and environment variables match the prior Python behaviour.
